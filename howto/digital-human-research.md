# 數位人方案研究報告

## 任務目標
研究並實作「好看 + 動作 + 對口型」的全身數位人解決方案。

---

##付費 API 方案

### HeyGen API

| 項目 | 內容 |
|------|------|
| **官網** | https://www.heygen.com/ |
| **免費額度** | 每月 3 支影片（1分鐘/720p） |
| **付費** | $0.05/秒（Photo Avatar），Video Agent $0.033/秒 |
| **對口型** | ✅ 優秀（Avatar IV/V） |
| **全身動作** | ⚠️ 有限（主要是上半身手勢，無全身移動） |
| **即時性** | ❌ 非同步，生成需數秒到數分鐘 |
| **輸出** | MP4/WebM，支援 720p/1080p/4K |

**評價：**
- 對話頭品質優秀
- 不適合即時對話
- API 可與 AI Agent 整合（MCP）
- 成本：60秒影片約 $3-6

### D-ID API

| 項目 | 內容 |
|------|------|
| **官網** | https://www.d-id.com/ |
| **免費額度** | 有限（測試用） |
| **付費** | $5.9/月（40 credits）起 |
| **對口型** | ✅ 優秀 |
| **全身動作** | ❌ 只有上半身（V3 Pro 有輕微肢體動作） |
| **即時性** | ⚠️ 有 Real-Time Agent（100 FPS WebRTC）但僅上半身 |
| **輸出** | MP4，最長 5 分鐘 |

**評價：**
- 有 Real-Time 串流模式（WebRTC）
- 專為對話設計，内建 LLM 整合
- 成本中等

## 開源本地方案

### SadTalker (OpenTalker)

| 項目 | 內容 |
|------|------|
| **GitHub** | https://github.com/OpenTalker/SadTalker |
| **功能** | 單張照片 + 音頻 → 說話頭部影片 |
| **對口型** | ✅ 優秀（CVPR 2023） |
| **輸出** | 臉部或全身（--still 模式） |
| **macOS M2** | ⚠️ 可用（CPU/MPS），但非常慢（非即時） |
| **即時性** | ❌ 批次處理，生成一分鐘需數分鐘 |
| **ComfyUI** | ❌ 無（但有 SD WebUI 擴展） |

**評價：**
- 唯一能在 M2 Mac 上跑的開源對口型方案
- 速度慢，不適合即時
- 適合離線預先生成

### MuseTalk (TMElyralab)

| 項目 | 內容 |
|------|------|
| **GitHub** | https://github.com/TMElyralab/MuseTalk |
| **功能** | 影片 + 音頻 → 對口型影片 |
| **對口型** | ✅ 優秀 |
| **輸出** | 256x256 半身 |
| **macOS M2** | ❌ 不支援（需要 CUDA） |
| **即時性** | ✅ 可以（在 NVIDIA V100 上 30fps） |
| **ComfyUI** | ✅ 有第三方整合 |

**評價：**
- 速度快，品質高
- **完全不支援 macOS**
- 需要 NVIDIA GPU

## 評估結論

### 短期方案（立即可用）
1. **MP4 模式** - 說話時播放預錄真人影片，即時但無對口型
2. **HeyGen/D-ID API** - 付費但功能完整，適合離線預先生成

### 長期方案（開源本地）
1. **SadTalker** - 唯一能在 M2 Mac 跑的方案，但速度慢
2. **MuseTalk** - 需要 NVIDIA GPU，效能佳

### 最佳組合建議
- **即時對話** → MP4 模式（犧牲對口型換取即時性）
- **預先內容** → HeyGen API（高品質離線生成）
- **實驗用途** → SadTalker（本地對口型研究）
